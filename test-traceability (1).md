# Requirements to Test Traceability – Student Task Manager

## Overview

This document maps each system requirement from the SRS to the corresponding test case(s) that verify it. This ensures every requirement is covered by at least one test and that no functionality goes unverified.

---

## Traceability Table

| Requirement ID | Requirement Description | Test Case |
|---|---|---|
| R1 | User can register for an account | TC-01 |
| R2 | Email address must be in a valid format | TC-02 |
| R3 | Required fields must be completed before form submission | TC-03 |
| R4 | User can create a new task with a name, due date, and priority level | TC-04 |
| R5 | Due dates must be today or a future date | TC-05 |
| R6 | User can mark a task as complete | TC-06 |
| R7 | User can delete a task | TC-07 |
| R8 | Only authenticated users can access the dashboard and task data | TC-08 |

---

## Notes

Every functional requirement defined in the SRS has been assigned at least one test case. This traceability matrix will be updated as new requirements or test cases are added during future development iterations.
